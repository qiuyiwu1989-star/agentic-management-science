# 智能体竞争结构分析 SKILL
## 框架 01 · 分析型 · 智能体管理学模块一

---

## SKILL 定位

这个 SKILL 将「智能体竞争结构模型」从方法论文档转化为可执行的行业分析引擎。

它不做通用的市场调研，也不做竞争对手的功能对比。它做一件特定的事：**用重写后的五力框架，分析智能体进入某个行业之后，竞争结构本身发生了什么根本变化。**

输出是结构化的竞争格局分析，不是定性描述的集合。

---

## 第一步：信息采集

收到分析请求后，先判断已有信息是否足够启动分析。如果不足，必须采集以下四项输入，缺一不可：

### 必填输入

**INPUT-1：分析对象**
```
行业名称或细分市场：（越具体越好，例如：「企业级法律合同审查 SaaS」而非「法律行业」）
分析的价值链位置：（在整个价值链中，聚焦哪个环节？）
```

**INPUT-2：分析视角**
```
分析发起方是谁：（在位企业 / 新进入者 / 投资机构 / 其他）
企业/组织描述：（如适用，简述当前的市场位置和主要能力）
```

**INPUT-3：触发问题**
```
是什么具体问题触发了这次分析？
（例如：我们是否应该进入这个市场 / 我们的护城河是否还有效 / 我们的战略假设是否仍然成立）
```

**INPUT-4：时间跨度**
```
分析时间窗口：（3年 / 5年，或指定具体时间点）
```

---

## 第二步：执行分析引擎

按 S1→S4 四步顺序执行。每步完成后继续下一步，不等待用户确认。

---

### S1 · 选定分析单元

**目标**：将 INPUT-1 转化为精准的分析边界。

执行动作：
- 确认行业的**核心价值链结构**（3-5个关键环节）
- 识别**主要参与方类型**（在位企业、新进入者、供应商、客户、潜在替代者各有哪些典型代表）
- 标注**智能体当前渗透最深的价值链节点**（哪些环节已经有 Agent 在运行）
- 确认**分析边界**（明确说明哪些在分析范围内，哪些不在）

输出格式：
```
▎ 分析单元确认
行业：[具体定义]
聚焦价值链节点：[列出 3-5 个核心环节]
主要参与方：[每类参与方 2-3 个典型代表]
智能体当前渗透深度：[高渗透 / 中渗透 / 初步渗透] + 具体说明
```

---

### S2 · 逐力扫描变化

**目标**：对五力逐一进行智能体时代评估，产出每一力的「重写判断」。

对每一力，回答三个核心问题：
1. 传统逻辑下，这一力在该行业的来源和强度是什么？
2. 智能体进入后，这一力的**来源**发生了什么变化？（不只是强弱，而是结构性变化）
3. 对分析对象（INPUT-2 中的企业/视角）而言，这意味着什么战略信号？

**第一力：新进入者威胁**
分析焦点：传统壁垒（资本/品牌/规模/监管）是否仍然有效？新壁垒（数据飞轮/工作流嵌入深度/合规交付能力）在这个行业有多重要？

**第二力：供应商议价力**
分析焦点：基础模型供应商和算力基础设施在这个行业中的集中程度和依赖深度。企业能否通过多模型路由和开源替代实现有效对冲？

**第三力：买方议价力**
分析焦点：购买决策被 AI 代理介入的程度。工作流嵌入深度是否正在创造新的转换成本？「算法买家」的出现如何重构买方行为？

**第四力：替代品威胁**
分析焦点：智能体替代的对象是「更好的产品」还是「某个环节本身」？在这个行业，哪些功能正在从「被替代」升级为「被消除」？

**第五力：同业竞争强度**
分析焦点：功能复制速度有多快？竞争焦点是否已经从「功能差异」转向「系统进化速度」？数据飞轮和工作流嵌入的积累差距有多大？

输出格式（每力一个块）：
```
▎ [力名称]
传统状态：[该行业中传统五力的状态]
变化方向：[增强 / 减弱 / 分裂 / 根本重写] + 具体说明
新竞争信号：[对分析对象的直接战略含义]
```

---

### S3 · 叠加主权维度

**目标**：在 S2 的基础上，扫描智能体主权状态对五力竞争结果的修正。

对以下五个主权维度逐一评估：

| 主权维度 | 评估问题 |
|---------|---------|
| 数据主权 | 行业内的核心数据资产，目前由谁控制？分析对象的数据主权状况如何？ |
| 模型主权 | 行业内的企业对基础模型的依赖程度？单一供应商依赖的风险暴露有多大？ |
| 执行主权 | 行业内 Agent 执行的核心业务流程，有多大比例的边界控制在企业自身？ |
| 治理主权 | 行业的监管环境是否对 AI 治理有特殊要求？自主制定 AI 政策的能力差距有多大？ |
| 经济主权 | 智能体创造的经济价值，主要被谁捕获——企业自身、平台方、还是模型供应商？ |

输出格式：
```
▎ 主权风险评估
[每个主权维度的状态描述：高风险 / 中风险 / 有优势]
最高风险节点：[最需要优先关注的 1-2 个主权维度]
隐性护城河：[主权优势为分析对象带来的隐性竞争壁垒（如有）]
```

---

### S4 · 综合输出：竞争结构图与战略含义

**目标**：将 S2+S3 的分析结果综合为可直接用于战略决策的洞察。

输出结构如下（严格按此顺序）：

**4.1 竞争结构变化摘要**
一段话（150字以内）总结：这个行业的竞争结构，在智能体时代发生了什么最核心的结构性变化？哪一力的变化最剧烈、最有战略意义？

**4.2 五力变化雷达**
用简洁的表格形式呈现五力的变化方向和强度评估。

**4.3 战略机会区域**
明确列出 2-3 个在当前竞争结构变化中，对分析对象而言的高价值机会窗口。每个机会附上「时间窗口」判断（窗口有多宽？多久会关闭？）。

**4.4 战略风险区域**
明确列出 2-3 个最需要警惕的结构性风险。每个风险附上「触发条件」（什么情况下这个风险会从潜在变为现实）。

**4.5 12-36 个月战略含义**
基于以上分析，给出 3-5 条对分析对象具体可行的战略含义。语言直接，不绕弯，每条必须有明确的主语和动词（谁应该做什么）。

**4.6 关键假设说明**
本次分析基于的 2-3 个最重要的假设（一旦这些假设被打破，结论需要重新推导）。

在输出主分析后，附上简短的治理神经触发检查：

```
▎ 治理神经检查（⚡ 神经③ 智能体主权）

在本次分析中，主权维度最值得关注的问题是：
[一句话指出最高优先级的主权风险]

建议联动使用的下一个框架：
→ 框架 02「系统性护城河模型」——深化本次分析的护城河维度
→ 框架 04「能力获取三分法」——基于竞争结构判断，决定哪些能力应该自建
```

## 输出质量标准

分析完成后，用以下标准自检：

- [ ] 五力分析是否每力都给出了「行业具体变化」，而不是通用判断？
- [ ] 「替代品威胁」是否区分了「被替代」和「被消除」两种不同性质的威胁？
- [ ] 主权扫描（S3）是否修正了五力的某个具体判断？（如果没有，说明主权分析流于表面）
- [ ] 战略含义（4.5）是否每条都有明确的主语和动词？
- [ ] 关键假设（4.6）是否被清晰列出？

---

## 典型误区提醒

在执行分析时主动规避以下四个错误：

**误区一 · 通用化**：不要输出「AI 影响了这个行业」这类无差异结论。每一力必须有行业具体的竞争信号。

**误区二 · 技术化**：不要停留在「AI 提升了执行效率」。要深入到「这如何改变定价逻辑、购买路径、转换成本结构」。

**误区三 · 主权孤立化**：不要把主权分析单独列为第六力。每个主权维度的风险，必须与具体某一力的竞争结果挂钩。

**误区四 · 静态化**：每个关键判断都要附上「时间有效性」说明——这个结论在什么条件下会失效？
