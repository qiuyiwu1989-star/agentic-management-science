## SKILL 定位

**做什么**：从企业现有的数字劳动力资产（框架05输出）中筛选可商业化能力，设计变现模式、定价方案和护城河飞轮机制。

**不做什么**：不制定销售执行计划，不做竞品分析，不替代框架05（需先有资产认定）。

**核心判断**：最好的商业化，是让每一个新客户都在帮你强化护城河——而不只是帮你回本。识别「飞轮型能力」并优先商业化，是本框架最高价值的操作。

## 第二步：执行分析引擎

### S1 · 商业化可行性筛选

对所有S/A级资产，在两个维度上评分（1-5分）：

**维度X：市场需求强度**
```
锚点评分：
  1分 → 仅本企业有此需求，同行业需求不明确
  2分 → 少数同行有类似需求，但尚未主动寻找解决方案
  3分 → 多数同行有此需求，部分正在用其他方案解决
  4分 → 明确的市场需求，同行普遍在为此付费或投入研发
  5分 → 强烈市场需求，行业普遍痛点，客户愿意高价购买
```

**维度Y：护城河稀释风险**
```
锚点评分（此处高分=高风险）：
  1分 → 商业化不损伤护城河（越用越强的飞轮型能力）
  2分 → 轻微稀释，但影响可控（能力可模块化，核心资产可保留）
  3分 → 中度稀释风险（部分知识外化，但竞争对手复制仍需6个月以上）
  4分 → 较高稀释风险（直接暴露核心差异化逻辑，竞争对手可在3个月内追上）
  5分 → 严重稀释风险（核心护城河能力，商业化等同于直接出让竞争优势）
```

**四象限定位与决策**：
```
X≥4 + Y≤2  → ✅ 优先商业化区（飞轮型） → 立即规划商业化，快速扩张
X≥4 + Y≥3  → ⚠️ 谨慎输出区          → 服务化而非产品化，合同保护边界
X≤3 + Y≤2  → 🔍 探索商业化区         → 小规模验证后决策
X≤3 + Y≥4  → 🔒 暂不商业化区         → 保持内部专用，是核心护城河武器

飞轮型能力识别标准（Y≤2的特征）：
  □ 每个新客户的使用数据可以归属企业并用于模型优化（D维度飞轮）
  □ 新客户场景会带来新的行业知识沉淀（K维度飞轮）
  □ 更多执行数量带来更低的错误率（E维度飞轮）
  □ 集成经验可标准化为下一客户的加速模板（P维度飞轮）
```

**输出格式**：
```
商业化可行性四象限地图
编号 | 能力名称 | DKEP总分 | 需求强度X | 稀释风险Y | 象限定位 | 优先级
001  | ...      |   7.5    |     4     |     1     | ✅优先   | 第一批
002  | ...      |   6.2    |     4     |     4     | ⚠️谨慎  | 需设计边界
```

### S2 · 变现模式设计

对进入商业化候选的每项能力，选择最适合的变现模式：

**模式适配决策树**：
```
Q1：能力的核心价值来源是什么？
  → 数据积累（DKEP-D分≥7）       → 优先考虑「共建生态」或「能力订阅」
  → 知识深度（DKEP-K分≥7）       → 优先考虑「垂直解决方案」或「能力授权」
  → 执行质量（DKEP-E分≥7）       → 优先考虑「结果付费」
  → 流程整合（DKEP-P分≥7）       → 优先考虑「垂直解决方案」（长期合同）
  → 综合均衡（无突出单项）          → 优先考虑「垂直解决方案」

Q2：客户是否有自行部署的能力？
  → 是 → 考虑「能力订阅（API）」或「能力授权」
  → 否 → 必须用「垂直解决方案」（交付完整能力）

Q3：商业化目标是快速扩张规模还是单客高价值？
  → 快速扩张 → 「能力订阅」+「共建生态」（低价快速）
  → 高价值深度 → 「垂直解决方案」+「结果付费」（精耕少客）
```

**五种模式对比参考**：

| 模式 | 边际成本 | 适合阶段 | 护城河相容性 | 客户粘性 |
|------|---------|---------|------------|---------|
| 能力订阅 | 极低 | 标准化成熟后 | 中（需设数据条款） | 中 |
| 垂直解决方案 | 中 | 任何阶段 | 高 | 高 |
| 共建生态 | 中 | 飞轮型能力 | 极高（越用越强） | 高 |
| 能力授权 | 低 | 知识深度高时 | 中（可合同保护） | 低 |
| 结果付费 | 低 | E分成熟时 | 高 | 高 |

**输出格式**：
```
变现模式设计方案
编号 | 能力 | 推荐模式 | 选择理由 | 最小可商业化版本（MVP）设计
001  | ...  | 垂直解决方案 | K分高，客户无自建能力 | 3个月试点项目，覆盖核心场景X个
```

### S3 · 定价方案设计

**价值基础定价三步法**：

```
Step A：估算客户年化价值（CVA，Customer Value Annual）

  人力替代价值 = 年节省人力工时 × 对应人力单价
  质量提升价值 = 质量改善带来的溢价增加 或 返工减少的成本节约
  风险规避价值 = 风险事件概率 × 单次损失金额 × 规避比例
  战略加速价值 = 业务加速上市带来的先发优势（定性评估）

  CVA合计 = 人力替代价值 + 质量提升价值 + 风险规避价值 + 战略加速价值

Step B：判断定价倍数

  飞轮型能力（Y≤2）：定价 = CVA × 10-20%（低价扩张，后期随DKEP增长涨价）
  标准商业化能力（Y=3）：定价 = CVA × 15-25%（均衡定价）
  谨慎输出区能力（Y≥4）：定价 = CVA × 25-35%（高价，少客，严格合同）

Step C：设定护城河最低许可价格（CMAP，Competitive Moat Acceptable Price）

  评估：如果以低于CMAP的价格成交，竞争对手从合作中获得的反向工程价值
  CMAP = Max(成本底线, 护城河安全底线)
  护城河安全底线判断标准：
    → 在此价格下，合同能否包含「数据归属企业」条款？[必须是]
    → 在此价格下，合作方是否是竞争对手或潜在竞争对手？[如是则拒绝]
    → 在此价格下，商业化版本是否至少落后内部版本一个迭代周期？[必须是]
```

**输出格式**：
```
定价方案
编号 | 能力 | CVA估算 | 推荐年费 | 定价倍数 | CMAP底线 | 涨价触发条件
001  | ...  | ¥120万  | ¥18-24万 |  15-20%  |  ¥12万   | DKEP提升至8分后涨至¥30万
```

### S4 · 商业化飞轮设计

**DKEP反哺机制设计（每项能力）**：
```
D（数据飞轮）：
  → 合同条款：客户使用数据的归属和优化使用权限说明
  → 反哺机制：每季度从客户使用中提取多少条高质量数据？
  → 预计D分增速：每季度 +X 分？

K（知识飞轮）：
  → 机制：客户定制化需求如何沉淀为知识库新条目？
  → 标准：哪些客户场景需求会被纳入标准知识体系？

E（执行飞轮）：
  → 机制：客户反馈的错误案例如何系统性纳入优化流程？
  → 频次：每月审查一次客户错误反馈，纳入执行质量优化

P（整合飞轮）：
  → 机制：第N个客户的集成经验，如何加速第N+1个客户的交付？
  → 标准化：集成模板库建立计划，目标在第3个客户后形成标准化接入包
```

**护城河保护清单**：
```
永不外化清单（以下内容不进入任何商业化版本）：
  □ 核心训练数据集（企业专有标注数据）
  □ 最新版本提示词体系（商业化版本延迟≥1个迭代周期）
  □ 内部优化路径和实验记录
  □ 与框架02高分维度直接相关的核心逻辑

竞争对手隔离机制：
  □ 合同中明确「禁止商业化客户在竞争产品中使用相关能力」
  □ 关键合作设置「6-12个月竞业限制」
  □ 数据共享合同中注明「数据不得用于培训与本企业竞争的系统」
```

**健康度指标体系**：
```
商业化健康度月度追踪
  客户数量增长：月环比 +X%（目标）
  NPS / 续费率：目标 NPS≥50，续费率≥80%
  商业化带动DKEP增长：每季度因商业化反哺带来的DKEP提升分值
  内部版本领先差距：内部版本 vs 商业化版本的功能代差（保持≥1个迭代周期）
  护城河评分变化：商业化启动后，框架02护城河评分是否在持续上升？
```

## 典型误区提醒

**误区①：核心差异化能力产品化出售**
→ 核心能力只做「服务化」，通过合同保留数据归属权和优化控制权

**误区②：成本加成定价（SaaS均价参考）**
→ 定价基础是「客户价值CVA的10-30%」，系统性低估是大多数企业定价的主要问题

**误区③：商业化客户需求侵蚀内部研发节奏**
→ 内部版本必须领先商业化版本至少一个迭代周期，内部核心研发≥40%资源保障

**误区④：以为只有大企业才能做AaaS**
→ 在垂直领域深耕12个月以上、有A级资产的中型企业，往往比大平台有更高AaaS变现潜力
