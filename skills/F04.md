## SKILL 定位

**做什么**：对企业需要引入的智能体能力，逐一判断自建（Build）/ 采购（Buy）/ 编排（Orchestrate）的最优路径，并给出有依据的决策表。

**不做什么**：不评估具体供应商优劣，不生成研发计划，不替代框架02/03的前置分析。

**核心判断**：能力获取路径选择本质上是主权决策——每次选路径，都在划定企业的「能力主权边界」，而不只是解决效率问题。

## 第二步：执行分析引擎

### S1 · 能力清单化与优先排序

对 INPUT-1 进行结构化整理，输出：

```
能力需求清单（结构化版）
编号 | 能力描述（一句话）| 优先级 | 时间要求 | 隐性需求补充
-----|------------------|--------|----------|-------------
001  | ...              | 必须    | 6个月内  | ...
002  | ...              | 重要    | 12个月内 | ...
```

> 关键动作：检查是否有「隐性能力需求」——用户列出的往往是功能，背后可能有尚未显式化的能力依赖（如：「智能客服」背后依赖「知识库检索能力」+「对话管理能力」，应分拆）

---

### S2 · 三维评分与路径初判

对每项能力，评估三个维度并打分：

| 维度 | 高（3分） | 中（2分） | 低（1分） |
|------|----------|----------|----------|
| **A. 战略重要性** | 直接支撑核心护城河，竞争差异化的来源 | 支撑性能力，不直接构成差异化 | 通用功能，大量可替代供给 |
| **B. 构建可行性** | 内部有积累，12个月内可达可用水平 | 需从零起步，18-24个月可达 | 内部缺乏基础，市场有成熟供给 |
| **C. 市场成熟度** | 多家成熟供应商，API稳定，价格合理 | 供给存在但质量参差，切换成本较高 | 市场尚未成熟，无可信赖外部供给 |

**路径初判规则（按优先级排）**：
```
A=3 + B=3           → Build 优先
A=3 + B≤2 + C=3    → 先 Buy，制定12-18个月迁移计划
A=3 + B≤2 + C≤2    → Build（即便困难也需自建，主权不可让渡）
A=2 + C=3           → Buy 或 Orchestrate（视多能力协同需求）
A=2 + 多能力联动     → Orchestrate 优先
A=1 + C≥2           → Buy（快速获取，不占战略资源）
多能力协同价值 > 单点 → Orchestrate（无论A分值）
```

输出格式：
```
三维评分与路径初判表
编号 | 能力 | A战略 | B可行 | C市场 | 初判路径 | 初判理由
-----|------|-------|-------|-------|----------|--------
001  | ...  |  3    |  2    |  3    | 先Buy→迁移 | A高但B低...
```

## 第三步：治理神经③检查（智能体主权模型）

激活以下主权状态检查：

```
主权状态审查（框架04配套）

【完全主权区】Build路径能力清单
  → 数据：是否100%归属企业？ [Y/N]
  → 逻辑：核心算法/提示词是否可控迭代？ [Y/N]
  → 优化路径：是否有独立的模型优化能力？ [Y/N]

【受限主权区】Buy路径能力清单
  → 是否存在单点依赖（核心能力仅一家供应商）？ [Y/N → 若Y，标高风险]
  → 是否有合同保障：数据不被供应商使用、接口不被单方关闭？ [Y/N]
  → 是否有6-12个月的迁移应急预案？ [Y/N]

【架构主权区】Orchestrate路径
  → 编排层是否由企业自有团队设计和维护？ [Y/N]
  → 当任意一个接入能力被替换时，架构是否可以低成本切换？ [Y/N]
  → 编排层的核心数据（流转数据、优化记录）是否由企业掌控？ [Y/N]

主权健康度综合判断：
  健康（所有高风险项均为Y）/ 
  需关注（1-2个N，已有缓解计划）/ 
  高风险（核心维度主权旁落且无应对预案）
```

## 典型误区提醒

**误区①：把编排当折中**
→ Orchestrate 必须有独立战略理由（多能力协同价值），不是「不想Build也不想Buy」时的默认选项

**误区②：用成本优化合理化核心能力采购**
→ 成本比较必须纳入「供应商停服/被收购的真实损失」，这个隐性成本通常被10倍低估

**误区③：什么都想Build**
→ 严格区分「对我们重要」与「必须由我们自建」——竞争对手也能买到的能力，自建不产生差异化

**误区④：路径决策一次定终身**
→ 每12个月必须复审：Buy路径供应商是否仍可信？Orchestrate复杂度是否到内化临界点？
