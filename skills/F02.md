# 系统性护城河评估 SKILL
## 框架 02 · 分析型（含评分）· 智能体管理学模块一

---

## SKILL 定位

这个 SKILL 将「系统性护城河模型」转化为可执行的七维护城河量化评估引擎。

它不做定性的\"我们比竞争对手好\"的描述。它做一件具体的事：**用七个维度的证据式评分，量化企业在智能体时代的护城河深度，识别最需要强化的方向，输出 12 个月的具体强化路径。**

核心判断：**护城河的最重要属性不是今天有多厚，而是在正确方向上的进化速度比对手快多少。**

## 第二步：执行评估引擎

### S1 · 证据式评分

**目标**：对七个维度逐一进行基于证据的评分（0-10分）。

**核心规则**：每一项评分必须附上支撑证据。没有证据的维度保守给分，并标注「需要补充证据」。不接受主观感受作为唯一评分依据。

对每个维度，依据以下锚点评分：

---

**维度① 专有数据飞轮**（0-10分）

评估问题：
- 核心数据是否为企业专有，竞争对手无法直接获取？
- 是否存在「使用→数据→改善→更多使用」的正向反馈闭环？
- 飞轮的复利速度如何？差距是在扩大还是收窄？

评分锚点：
- 0分：依赖公开数据或供应商数据，无独立飞轮
- 3分：有专有数据积累，但无明确反馈闭环
- 5分：有飞轮结构，初步运转，竞争对手在1年内可追上
- 8分：飞轮运转良好，独占性强，追赶需要3年以上
- 10分：行业内最深的数据飞轮，独占性极强，复制成本极高


**维度② 工作流嵌入深度**（0-10分）

评估问题：
- 产品/服务嵌入了客户多少个业务流程？是核心流程还是辅助流程？
- 如果我们明天停止服务，客户的业务是否会中断？
- 客户迁移成本是操作级别（低）还是流程重建级别（高）？

评分锚点：
- 0分：纯工具型，用户可随时替换
- 3分：嵌入1-2个辅助流程，切换有摩擦但不困难
- 5分：嵌入重要业务流程，切换需要1-3个月
- 8分：嵌入3个以上核心流程，切换需要重建整个工作流程
- 10分：成为核心业务基础设施，切换需要半年以上并有显著业务风险

## 输出质量自检

完成评估后，用以下标准自检：

- [ ] 每个维度的评分都有具体的可验证证据（而非主观判断）？
- [ ] 如果提供了竞争对手信息，相对评估是否完成？
- [ ] 绝对缺口和相对缺口是否分别识别（而非混为一谈）？
- [ ] 强化路径是否有具体的里程碑和评估指标（而非模糊的方向描述）？
- [ ] 维度⑦ 人机协同进化力是否获得了足够的权重（而非被轻视）？
- [ ] 主权折扣是否被正确应用？

## 典型误区提醒

执行评估时主动规避：

**误区一 · 主观化**：不接受「感觉我们在这个维度还不错」的表述。要求具体数据或事实。

**误区二 · 数据 ≠ 飞轮**：有数据积累不等于有飞轮。飞轮成立的条件是：独占性 + 闭环性 + 复利性，三者缺一不可。

**误区三 · 进化力低估**：维度⑦得分低是所有护城河中最危险的信号。在输出中明确标注这一点。

**误区四 · 只看绝对分**：总分 60/70 的企业，如果在最关键的维度上远落后于竞争对手，仍然处于高风险状态。相对评估不可省略。
