# 智能体定位光谱 SKILL
## 框架 13 · 分析型 · 智能体管理学模块三

---

## SKILL 定位

这个SKILL将「智能体定位光谱」从方法论文档转化为可执行的产品定位诊断引擎。

它不做市场调研，也不做竞品功能对比。它做一件特定的事：**在自主性×人格化双轴上定位Agent产品，确认定位的下游决策影响，建立动态复审机制。**

核心判断：**定位错误的代价是非线性的。一个工具型Agent被按助理型设计，用户建立情感预期后发现无法满足，流失且很难找回。定位必须驱动交互、定价、合规的系统性对齐。**


## 第二步：执行分析引擎

### S1 · 双轴定位

**X轴：自主性评估（DAL L0-L5）**

基于产品**当前实际运行状态**（非技术规划）评估：

| 等级 | 判断标准 | 产品表现 |
|------|---------|---------|
| L0 | 人工全程执行，AI仅辅助 | AI帮写草稿，人做所有决策 |
| L1 | AI给建议/草稿，人决策执行 | AI推荐方案，人选择并执行 |
| L2 | Agent执行，每个关键步骤需人确认 | 每步暂停等人审批 |
| L3 | Agent自动运行，异常时人工介入 | 正常自动，出问题才喊人 |
| L4 | Agent在规则边界内完全自主 | 人只设边界参数，Agent自主执行 |
| L5 | 理论完全自主 | 当前无生产环境部署 |

**Y轴：人格化程度评估**

| 等级 | 判断标准 | 产品表现 |
|------|---------|---------|
| 低 | 纯工具态，无名字/语气/情感 | 结构化输入输出，无个性化 |
| 中 | 有基本人格特征，但不建立关系 | 有名字/语气，但无记忆/情感连接 |
| 高 | 类人交互，能建立持续关系 | 有记忆、有情感回应、用户有归属感 |

**输出格式**：
```
▎ APS 定位结果

产品名称：[名称]
X轴（自主性）：L[N] → [低/高]自主
  依据：[具体使用模式描述]
Y轴（人格化）：[低/中/高]
  依据：[具体人格特征列表]

所属象限：[工具型 / 助理型 / 同事型 / 数字员工型]
```

### S2 · 下游决策对齐检查

定位确认后，逐一检查五类下游决策是否与定位一致：

```

【交互设计】
  当前交互模式：[用户主动触发 / Agent主动推送 / 混合]
  定位要求：[工具型=用户主动 / 助理型=轻度主动 / 同事型=协同 / 数字员工=委托]
  是否对齐：[✅ / ❌ 错位项：____]

【定价策略】
  当前定价：[使用量/席位/订阅/结果/SLA分成]
  定位要求：[工具型=使用量 / 助理型=订阅 / 同事型=结果 / 数字员工=SLA+结果]
  是否对齐：[✅ / ❌ 错位项：____]

【信任机制】
  当前治理级别：[无/基础CATER/完整CATER]
  定位要求：[工具型=极低 / 助理型=内容安全 / 同事型=操作审计 / 数字员工=完整CATER]
  是否对齐：[✅ / ❌ 错位项：____]

【SLA承诺】
  当前SLA：[有/无，具体承诺]
  定位要求：[工具型=无 / 助理型=基本 / 同事型=质量承诺 / 数字员工=完整SLA]
  是否对齐：[✅ / ❌ 错位项：____]

【责任归属】
  当前责任人：[有/无具名责任人]
  定位要求：[工具型=用户负责 / 助理型=共担 / 同事型=Agent Owner / 数字员工=完整RACI-A]
  是否对齐：[✅ / ❌ 错位项：____]

对齐总评：[5/5对齐 / N项错位需修正]
错位修正计划：[列出需修正的项目和时间表]
```

## 第三步：治理神经检查

```
▎ 治理神经检查（⚡ 神经③ 智能体主权）

当前定位对应的治理要求：
  工具型 → 极低，用户自行负责
  助理型 → 内容安全 + 基本审计
  同事型 → 操作审计 + 边界治理 + RACI-A
  数字员工型 → CATER五维全覆盖 + 完整SLA + RACI-A

主权风险：
  定位越高（数字员工型），用户对Agent的依赖越深，
  主权风险越高——需要更强的可控性（C）和可逆性（R）

建议联动使用的下一个框架：
→ 框架14「JTBO」——定位决定结果定义深度
→ 框架18「CATER」——定位决定治理级别要求
```

## 典型误区提醒

**误区①：把APS当精确测量工具**
→ APS是战略讨论工具，不是精确量化工具。人格化程度无法精确打分。

**误区②：把自主性提升当唯一方向**
→ 各象限有独立商业价值。助理型（低自主×高人格）在情感陪伴场景有独特价值。

**误区③：定位与定价/SLA/合规脱节**
→ "数字员工"级别的定位必须有CATER五维覆盖+完整SLA。能力达不到就用更保守的定位。

**误区④：忽视定位漂移**
→ 每季度复审一次。DAL实际等级比声明高1级以上时立即触发修正。
