## SKILL 定位

**做什么**：绘制业务流程的五节点任务流图，识别各类节点的职责与缺口，设计节点协调规则，评估管理者编排幅度。

**不做什么**：不替代框架08（具体架构分层），不做人员编制方案，不做具体Agent技术选型。

**核心判断**：组织设计的起点是任务流图，不是汇报关系图。五类节点（人类/Agent/工具/编排/治理）各有不可替代性，缺少任何一类都会导致网络结构性脆弱。

## 第二步：执行分析引擎

### S1 · 绘制任务流图

对每个目标流程，按步骤拆解并标注：

```
流程：[流程名称]

步骤 | 任务描述 | 当前执行主体 | 输入来源 | 输出去向 | 触发条件 | 失败处理
 1  | ...      | 人类/Agent  | ...      | ...      | ...      | 无/升级/重试
 2  | ...
```

**关键识别项**（逐步骤标注）：
- 哪些步骤已可由Agent承担，理由是什么？
- 哪些步骤必须由人类主导，不可替代的原因是什么？
- 哪些步骤当前无人/无系统监管（潜在孤儿节点）？

---

### S2 · 五节点地图

对每个核心流程，识别五类节点并标注缺口：

```
五节点地图 · [流程名称]

人类节点（Human Nodes）
  节点描述：[谁承担高判断/高关系/高模糊任务]
  当前职责：[列出]
  缺口：[是否有人类节点缺失导致某些高风险任务无人把关]

智能体节点（Agent Nodes）
  已有Agent：[名称/职责/自主程度]
  建议新增：[哪些任务适合Agent承担但尚未配置]
  缺口：[是否有Agent无明确任务边界在「自由运行」]

工具节点（Tool Nodes）
  已接入工具：[CRM/ERP/API等]
  缺失工具：[哪些能力缺口需要工具节点补充]

编排节点（Orchestration Nodes）
  当前编排机制：[谁/什么系统在做任务路由和调度]
  缺口：[是否存在任务在节点间传递无明确规则的环节]

治理节点（Governance Nodes）
  当前覆盖：[哪些环节有权限控制/审计/回滚]
  缺口：⚠️ [哪些Agent节点缺乏对应治理节点覆盖——这是最高优先级风险]

节点健康度评分（1-5分）：
  人类节点：X分 · Agent节点：X分 · 工具节点：X分
  编排节点：X分 · 治理节点：X分（治理节点低分需立即处理）
```

---

### S3 · 节点协调规则设计

为每个Agent节点和编排节点设计显式协调规则：

```
协调规则文档 · [流程/节点名称]

【升级规则】（Agent必须转交人类节点的条件）
  □ 任务类型超出已定义范围
  □ 输出置信度/质量低于 [具体阈值]
  □ 涉及 [具体敏感场景/高风险操作]
  □ 连续X次尝试未达到质量标准
  → 升级目标：[具体人类角色]，响应时效：[X小时内]

【失败路由预案】（编排节点处理任务失败的三种策略）
  重试策略：[何种失败触发重试，最多重试几次]
  降级策略：[重试失败后的降级处理方式]
  升级策略：[降级仍无法处理时的人工介入路径]

【治理触发条件】
  权限异常触发：[什么操作超出授权范围需立即冻结]
  审计要求触发：[哪些操作必须产生审计日志]
  回滚触发：[什么情况下需要回滚已执行操作]
```

## 第三步：网络结构完整性检查

```
Agentic Network完整性自检

□ 五类节点是否全部存在？
  人类节点 [✅/⚠️] · Agent节点 [✅/⚠️] · 工具节点 [✅/⚠️]
  编排节点 [✅/⚠️] · 治理节点 [✅/⚠️/❌（立即处理）]

□ 是否存在「孤儿Agent节点」（无治理节点覆盖）？
  → 若存在：列出名称，设为最高优先级风险

□ 所有升级规则是否显式化（不依赖Agent自行判断）？
  → 若有模糊规则：标注并要求在本周内明确化

□ 编排节点是否有完整的三种失败预案？
  → 只有重试而无降级/升级预案的编排节点：列出

□ 治理节点的回滚机制是否定期测试？
  → 建议每季度测试一次回滚机制有效性

网络健康度综合评级：
  健康 → 五类节点完整，协调规则显式化，无孤儿Agent
  需关注 → 编排/治理节点存在缺口，协调规则部分模糊
  高风险 → 治理节点严重缺失，Agent节点存在孤儿状态
```

## 输出质量自检

- [ ] 每个目标流程都有任务流图（以任务为粒度，非汇报关系）
- [ ] 五类节点全部识别，治理节点缺口已标注为最高优先级
- [ ] 每个Agent节点有显式升级规则（非模糊描述）
- [ ] 编排节点有三种失败预案（重试/降级/升级）
- [ ] 关键管理者完成编排幅度评估
- [ ] 网络完整性自检完成，健康度已评级

## 典型误区提醒

**误区①：只画汇报关系图，不画任务流图**
→ Agentic Network的分析起点是任务流转路径，汇报关系图无法揭示节点间协作结构

**误区②：把Agentic Network等同于「全自动化」**
→ 人类节点处理高判断/高关系/高风险任务，是不可压缩的网络组成部分

**误区③：忽视治理节点，当作合规装饰**
→ 治理节点是规模化运行的前提；Agent错误有复利效应，治理节点必须与业务节点同步设计

**误区④：把扁平化等同于Agentic Network**
→ 减少层级≠改变协调方式；没有编排节点支撑的扁平化只带来混乱
