03_ // key cross-references

关键引用链路

40 个框架不是平行展开的——它们之间有大量交叉引用。这里列出体系里最关键的7 条贯穿性链路。

链路一 · 战略闭环
F00 ECOT 提出"能力编排" → F01 五力重构定位外部世界 → F02 护城河评估自身优势 → F03 APM 资源配置 → F04 TICA 获取路径
链路二 · 组织闭环
F07 网络组织（形态）→ F08 双前台中台（架构）→ F10 岗位体系（人才）→ F12 Kotter-AI（文化）
链路三 · 产品闭环
F13 定位 → F14 JTBO 结果 → F15 CADM 上下文 → F16 D-PMF 验证 → F17 VERITAS / F18 CATER（评估）
链路四 · 系统工程闭环
F24 ADLC 生命周期 → F20 E³ 评测驱动 → F25 TSEM 三层评测 → F22 PRTD 调试 → F26 PRT 回归
链路五 · 价值/运营闭环
F27 AOMM 成熟度 → F28 3DSP 指标 → F29 DWPS 绩效 → F30 CPST 成本 → F31 Agent ROI → F32 SRAG 风险
链路六 · 人才闭环
F33 IACMM 等级 → F34 TSMM 任务迁移 → F35 FALA 能力结构 → F36 AC6 人类核心
★ 链路七 · 治理神经横切
G01 意图管理 横切 F10（编排者）/ F14（JTBO）/ F15（上下文）· G02 边界升级 横切 F09（RACI×DAL）/ F11（安全三系统）/ F29（DWPS 安全维）· G03 主权模型 横切 F02（护城河主权维）/ F04（TICA 自建判断）/ F06（AaaS）/ F09（RACI A 具名）/ F18（CATER 可控）/ F36（价值锚定）
三条治理神经各自横切多个模块，形成整套体系的"哲学骨架"。
